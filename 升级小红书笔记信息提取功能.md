对小红书 SDK 进行功能扩展与优化，实现以下两个核心功能模块：

1. 小红书笔记信息提取功能：

   - 功能描述：当输入小红书笔记 URL 链接时，SDK 应能够准确提取并结构化以下笔记信息字段：
     - 基础内容字段：标题、正文内容
     - 媒体资源字段：图片（支持多张图片获取，需特别处理并支持实况 live 图片格式）、视频资源
     - 互动数据字段：作者发布时间、点赞数量、收藏数量、评论数量
     - 内容属性字段：是否原创标识
   - 技术要求：确保所有提取的字段数据结构清晰、格式统一，便于其他项目直接调用和解析

2. 小红书博主信息及笔记集合提取功能：

   - 博主个人信息提取：当输入博主个人中心 URL 链接时，SDK 需提取并返回以下博主信息：

     - 个人资料：头像 URL、博主名称、小红书号、IP 属地、个人签名
     - 社交数据：关注数量、粉丝数量、获赞与收藏总数量、发布笔记总数
     - 特殊标识：是否存在置顶笔记

   - 博主笔记集合提取：
     - 功能要求：自动获取该博主发布的全部笔记信息，每条笔记的字段结构需与上述"小红书笔记信息提取功能"保持一致
     - 分页机制：实现分页获取策略，初始请求获取 10 条笔记，随后自动获取剩余笔记内容，避免触发小红书 API 请求限制
     - 完整性保证：确保只要输入有效的小红书博主 URL，SDK 能够自动、完整地获取该博主发布的所有笔记内容

整体技术要求：

- 输入验证：实现严格的 URL 格式验证，确保只有有效的小红书笔记或博主 URL 才能触发相应的信息提取流程
- 错误处理：添加完善的错误处理机制，对无效 URL、网络异常、数据解析失败等情况提供明确的错误提示
- 数据一致性：保证笔记信息在单独提取和作为博主笔记集合提取时的数据结构完全一致
- 反爬策略：实现请求频率控制和合理的延迟机制，降低被目标网站限制的风险
