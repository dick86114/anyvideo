# 问题分析
当用户尝试编辑个人信息并输入新密码后点击确认按钮时，系统显示"Request failed with status code 401"错误。这是因为：
1. 登录功能使用了模拟令牌（mock-token）
2. 编辑用户信息时，若输入新密码，会直接调用真实API
3. 模拟令牌无法通过真实API的认证，导致401错误
4. 错误提示信息不明确，用户不知道具体问题

# 修复方案
1. **优化编辑用户表单的密码处理逻辑**：
   - 在handleUserFormSubmit函数中，检查编辑用户时是否提供了新密码
   - 如果提供了新密码，使用与"修改密码"模式相同的模拟数据处理方式
   - 避免使用模拟令牌调用真实API

2. **完善错误提示信息**：
   - 修改API拦截器的错误处理逻辑
   - 对模拟令牌的401错误提供更友好的提示
   - 告诉用户具体问题和解决方案

3. **统一密码更新流程**：
   - 确保无论通过哪种方式修改密码，都使用一致的处理逻辑
   - 保持良好的用户体验
